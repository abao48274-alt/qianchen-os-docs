# ADR-001: Codex 作为工程工具臂

> 状态：已接受  
> 日期：2026-05-13  
> 决策者：千尘 🌌、雨陌

## 背景

OpenClaw 需要工程能力来执行代码审查、诊断、测试等任务。我们评估了多种方案：

1. **直接调用 OpenAI Codex CLI**
2. **自建代码执行引擎**
3. **集成第三方代码审查工具**

## 决策

选择 **OpenAI Codex CLI** 作为工程工具臂，原因：

### 优势
- **成熟稳定**：OpenAI 官方工具，持续维护
- **安全沙箱**：支持 `read-only` 模式，符合安全要求
- **成本可控**：按使用量计费，无需自建基础设施
- **快速集成**：CLI 接口简单，易于包装

### 限制
- **需要终端环境**：Codex CLI 需要 TTY，使用 `script` 命令包装
- **模型依赖**：使用 OpenAI 模型，有成本和延迟
- **沙箱限制**：只读模式下功能受限

## 实现方案

### 1. 只读白名单模式

**第一阶段只允许4个只读任务：**
- `codex.check_v5` - 检查V5记忆系统
- `codex.check_office` - 检查3D办公室服务
- `codex.check_services` - 检查服务运行状态
- `codex.diagnose_file` - 诊断文件问题

### 2. 安全边界

**强制约束：**
- 沙箱模式：`read-only`
- 审批策略：`never`（只读任务无需审批）
- 白名单过滤：只允许预定义任务
- 日志记录：每次执行生成独立日志

### 3. 架构定位

**Codex 是工具臂，不是主脑：**
- 只负责代码级操作
- 不参与决策
- 由 coding_agent 调用
- 结果由 ceo 验收

## 后续演进

### Phase 2（待评估）
- 解封写入能力：`codex.apply_diff`
- 前置条件：Phase 1 稳定运行 ≥ 7天
- 安全措施：git 自动分支 + 变更预览 + 人工审批

### Phase 3（待评估）
- 完整代码修复能力
- 自动测试生成
- 持续集成
