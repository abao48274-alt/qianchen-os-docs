# CEO 重要消息交付协议 v1

**版本**：1.0  
**生效日期**：2026-05-14  
**制定者**：千尘（CEO）  
**状态**：正式生效

---

## 1. 协议目的

解决信息传递中的三个关键问题：
1. `message_bus` 不是 CEO 主脑记忆（编码损坏、不自动写入长期记忆）
2. Telegram 不是长期记忆（只代表用户侧收到）
3. CLI 长中文参数容易编码损坏

确保重要消息可靠到达 CEO 主脑会话并沉淀为长期记忆。

---

## 2. 重要消息定义

以下类型消息必须使用本协议：

### 2.1 强制使用（重要消息）
- **Codex 审查方案**：架构审查、代码审查、安全审查
- **修复报告**：故障修复、问题解决、系统恢复
- **架构决策**：技术选型、架构变更、重大设计决策
- **事故复盘**：故障分析、根因分析、改进措施
- **验收报告**：功能验收、性能验收、安全验收
- **长期规则变更**：工作流程、协议、规范的变更

### 2.2 可选使用（中等消息）
- 进度更新
- 状态同步
- 简单通知

### 2.3 禁止使用本协议（简单消息）
- 问候消息
- 简单确认
- 状态查询

---

## 3. 禁止的传递方式

重要消息**禁止**只通过以下方式传递：
- `message_bus`（只做副本，不作为主交付）
- Telegram（只做通知，不作为记忆）
- CLI 长参数（容易编码损坏）
- 终端输出（不持久化）

---

## 4. 标准三段式流程

### 4.1 第一段：写入持久化报告文件
**执行者**：消息发送方（Codex、子Agent、工具等）

**要求**：
1. 文件路径：`qianchen-os-docs/reports/YYYY-MM-DD-[报告类型]-[来源].md`
2. 文件格式：标准 Markdown
3. 内容要求：
   - 完整中文方案
   - 具体证据（日志、截图、测试结果）
   - 任务拆分（如适用）
   - 验收标准
   - 风险分析

**示例路径**：
- `qianchen-os-docs/reports/2026-05-14-codex-super-review.md`
- `qianchen-os-docs/reports/2026-05-14-fault-recovery-report.md`

### 4.2 第二段：短消息通知 CEO 主会话
**执行者**：消息发送方

**要求**：
1. 使用 `openclaw agent --agent ceo --message "..." --json`
2. 消息格式：
   ```
   [消息类型] 已生成报告，请读取：[文件路径]
   ```
3. 消息长度：不超过 200 字符
4. 必须包含：消息类型 + 文件路径

**示例**：
```
Codex 审查方案已生成报告，请读取：qianchen-os-docs/reports/2026-05-14-codex-super-review.md
```

### 4.3 第三段：CEO 写入当天 memory
**执行者**：CEO（千尘）

**要求**：
1. 读取报告文件
2. 回复"已读取"
3. 在 `memory/YYYY-MM-DD.md` 追加摘要：
   - 消息类型
   - 核心结论（1-2 句）
   - 报告文件路径
   - 决策/行动（如适用）

**摘要格式**：
```markdown
### [时间] [消息类型] 已读取
**核心结论**：[1-2 句结论]
**报告路径**：`[文件路径]`
**决策/行动**：[如有]
```

## 5. 验收标准

### 5.1 必须满足（全部通过）
1. **CEO 回复已读取**：CEO 主会话有明确回复
2. **memory 有摘要和链接**：`memory/YYYY-MM-DD.md` 包含摘要和报告路径
3. **message_bus 有状态副本**：同一 task_id 的状态记录

### 5.2 验收方法
1. 检查 CEO session 记录
2. 检查 `memory/YYYY-MM-DD.md` 文件
3. 检查 `message_bus.json` 中的 task_id

---

## 6. 异常处理

### 6.1 报告文件不存在
- 记录为"原始报告文件缺失"
- 尝试从其他来源重建（如 message_bus 片段）
- 标注"信息不完整"

### 6.2 编码损坏
- 不尝试修复损坏的编码
- 记录为"原始消息编码损坏"
- 建立新的完整报告

### 6.3 CEO 未及时读取
- 等待 30 分钟后再次通知
- 如仍无响应，记录为"CEO 会话异常"
- 通过其他渠道（如 Telegram）提醒

---

## 7. 职责分工

### 7.1 CEO（千尘）
- 读取报告文件
- 回复"已读取"
- 写入 memory 摘要
- 做出决策（如需要）

### 7.2 子Agent/工具
- 生成完整报告文件
- 发送短消息通知
- 在 message_bus 写入状态副本

### 7.3 验收Agent（qa_agent）
- 验证三处证据
- 记录验收结果
- 报告异常情况
