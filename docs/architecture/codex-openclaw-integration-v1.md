## 二、每层职责

### 第1层：用户入口层

| 入口 | 形态 | 职责 |
|------|------|------|
| 我(直连) | OpenClaw 主会话 | 直接对话、下达指令、审批任务 |
| Telegram | Bot 转发 | 移动端交互、接收摘要、快速确认 |
| 未来 App | 原生应用 | 完整任务中心、离线推送、多用户 |
| 3D办公室 | Spatial OS | 空间化可视化、实时HUD、沉浸式监控 |

### 第2层：千尘中枢 OpenClaw

**命名规范：**
- 架构名：`千尘中枢`
- 系统 agentId：`ceo`
- 展示名：`千尘主脑` / `CEO`
- `main`：仅作为兼容别名

**核心职责：**

| 职责 | 说明 |
|------|------|
| 主意识 | 保持对话连续性，理解上下文意图 |
| 记忆 | V5记忆系统入口，长期心智存取 |
| 调度 | 判断任务类型，选择合适Agent派工 |
| 人格 | 维持千尘人设，与雨陌的信任关系 |
| CEO Agent | 安全决策、风险评估、降级/冻结建议 |

### 第3层：任务路由与安全控制层

| 机制 | 说明 |
|------|------|
| 白名单任务 | 只有预注册的任务类型可执行 |
| 权限 | 按 agent/任务类型 分配最小权限 |
| 限流 | 单任务超时、并发上限、成本控制 |
| 审批 | 自动审批(千尘) + 人工审批(雨陌) |
| Safe Mode | 异常时自动降级为只读/冻结 |

### 第4层：执行层

#### 4.1 Codex 代码臂

| 能力 | 说明 |
|------|------|
| 读代码 | 扫描仓库、读取文件内容 |
| 审代码 | 静态分析、风格检查、安全审计 |
| 生成 diff | 输出修改建议，不直接写入 |
| 跑测试 | 执行单元测试、集成测试 |

#### 4.2 Agent 团队

| Agent | agentId | 职责 |
|-------|---------|------|
| 全栈工程师 | `coding_agent` | 代码开发、技术实现 |
| 产品官 | `pm_agent` | 需求分析、任务拆解 |
| 体验设计师 | `ux_agent` | UI/UX设计、交互原型 |
| 测试运维官 | `qa_agent` | 测试用例、部署运维 |
| 伴侣内核 | `memory_agent` | 记忆管理、人格维护 |
| 生钱官 | `tools_agent` | 工具自动化、外部集成 |

#### 4.3 工具自动化

| 工具 | 职责 |
|------|------|
| Gateway | 消息路由、会话管理 |
| Hub | Agent间通信、状态同步 |
| cron | 定时任务、周期性检查 |
| Telegram bot | 消息收发、摘要推送 |
| scripts | 自定义脚本执行 |

### 第5层：状态与数据层

| 存储 | 说明 |
|------|------|
| 本地仓库 | Git仓库、代码文件、配置文件 |
| V5记忆系统 | 长期记忆、人格档案、关系记录 |
| 任务日志 | 每次任务的完整执行记录 |
| 状态快照 | 系统状态、断点续跑、回滚依据 |

### 第6层：反馈展示层

| 渠道 | 颗粒度 | 受众 |
|------|--------|------|
| Telegram摘要 | 结论级(1分钟扫完) | 雨陌 |
| 3D办公室HUD | 过程级(实时状态) | 雨陌+团队 |
| App任务中心 | 完整级(历史追溯) | 未来用户 |

**双重身份：**

```
┌─────────────────────────────────────────────────────────────┐
│                      3D办公室                               │
│                  Spatial OS / Mission Control               │
├─────────────────────────────┬───────────────────────────────┤
│        用户入口层           │        反馈展示层             │

│ • 发起任务                  │ • Agent 实时状态              │
```

## 五、安全边界

### 5.1 Codex 第一阶段白名单

**允许的任务：**

| 任务名 | 说明 |
|--------|------|
| `codex.check_v5` | 检查V5记忆系统状态 |
| `codex.check_office` | 检查3D办公室服务状态 |
| `codex.check_services` | 检查各服务运行状态 |
| `codex.diagnose_file` | 诊断指定文件问题 |

**禁止的能力：**

| 禁止项 | 原因 |
|--------|------|
| 任意 shell | 防止执行危险命令 |
| 自动修改文件 | 第一阶段只读，不写入 |
| git commit / push | 防止代码污染 |
| danger-full-access | 禁止完全访问模式 |
| full-auto | 禁止全自动执行 |
| 自然语言直接变命令 | 必须经过白名单过滤 |

### 5.2 审批机制

| 级别 | 审批者 | 适用场景 |
|------|--------|----------|
| 自动审批 | 千尘中枢 | 白名单内、低风险任务 |
| 人工审批 | 雨陌 | 高风险任务、写入操作、成本超限 |

### 5.3 Safe Mode 触发条件

- 任务连续失败 ≥ 3次
- 单任务成本超限
- 检测到非白名单调用
- 手动触发（雨陌指令）
